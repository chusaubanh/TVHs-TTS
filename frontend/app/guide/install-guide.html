Hướng dẫn cài đặt Thành Vinh Studio

Tài liệu này dành cho người không rành công nghệ. Bạn chỉ cần làm đúng theo phần dành cho hệ điều hành của mình.

Nếu không biết bắt đầu từ đâu

Windows: bấm install.bat, sau đó bấm start.bat. macOS: bấm install.command, sau đó bấm start.command. Nếu bị lỗi lặt vặt, chạy file repair tương ứng rồi mở lại app.

Trước khi cài Cài nhanh Khi bị lỗi Cho người hỗ trợ

1. Trước khi cài

Máy cần có Internet trong lần cài đầu tiên để tải phần mềm và model. Sau khi tải đủ, app có thể chạy local trên máy.

Git phải được cài trước khi tải project từ GitHub. Nếu bạn nhận project bằng file ZIP đã giải nén sẵn thì có thể bỏ qua lệnh git clone và mở thẳng thư mục project.

Git Phần mềm để tải source code từ GitHub về máy. Bắt buộc. Windows dùng Git for Windows. macOS có thể dùng Xcode Command Line Tools hoặc Homebrew.
Python Phần mềm để chạy backend AI. Nên dùng Python 3.10 trở lên. Script sẽ dùng uv để quản lý môi trường.
Node.js Phần mềm để chạy giao diện web. Nên dùng Node.js 20 trở lên.
eSpeak NG Thư viện hỗ trợ xử lý phát âm. Script sẽ cố gắng tự cài. Nếu thiếu, app sẽ báo trong màn hình health.
Model AI File nặng để tạo giọng nói. Không nằm sẵn trong GitHub. App hoặc script sẽ tải khi bạn đồng ý.
Lưu ý: Nếu Windows/macOS hỏi quyền chạy file, chọn cho phép. Đây là script cài đặt dependency cho project.

2. Cài nhanh theo hệ điều hành

Windows

Mở thư mục project, double-click theo thứ tự:

install.bat

start.bat

File install.bat sẽ cài dependency. File start.bat sẽ mở app.

macOS

Mở thư mục project, double-click theo thứ tự:

install.command

start.command

Nếu macOS chặn file, mở Terminal trong thư mục project và chạy lệnh bên dưới.

chmod +x install.command start.command repair.command install.sh start.sh repair.sh

Linux

Mở Terminal trong thư mục project và chạy:

chmod +x install.sh start.sh repair.sh
./install.sh
./start.sh
Sau khi chạy thành công: trình duyệt sẽ mở tại http://localhost:3000. Backend chạy tại http://localhost:8000.

3. Khi bị lỗi lặt vặt

Rất nhiều lỗi không phải do app hỏng, mà do lần chạy trước còn giữ port, thiếu dependency, hoặc Next.js còn file khóa. Hãy chạy repair trước.

Windows

Double-click:

repair.bat

Sau đó chạy lại start.bat.

macOS

Double-click:

repair.command

Sau đó chạy lại start.command.

Linux

Chạy trong Terminal:

./repair.sh
./start.sh
Lỗi: port 3000 hoặc 8000 đang được sử dụng

Nguyên nhân thường là app cũ chưa tắt hết. File repair sẽ tự tắt process đang chiếm port 3000, 3001 và 8000.

Lỗi: Unable to acquire lock trong thư mục .next

Đây là lỗi Next.js còn file khóa từ lần chạy trước. File repair sẽ xóa frontend/.next/dev/lock và cache dev liên quan.

Lỗi: No module named 'vieneu'

Python chưa cài đúng package tạo giọng. File repair sẽ cài lại vieneu==2.7.0 và sea-g2p==0.7.5.

Lỗi: Frontend dependencies not found

Máy chưa có frontend/node_modules. Chạy install hoặc repair để cài lại bằng npm ci.

Lỗi: thiếu model hoặc chưa tải model

Model AI không lưu trong GitHub vì file lớn. Bạn có thể tải bằng nút trong app hoặc dùng download.bat trên Windows.

4. Những lần sau mở app thế nào?

1

Mở đúng file start

Windows dùng start.bat, macOS dùng start.command, Linux dùng ./start.sh.

2

Đợi trình duyệt mở

Script sẽ tự mở app. Nếu không tự mở, vào http://localhost:3000.

3

Nếu lỗi, chạy repair

Không cần xóa lung tung. Chạy file repair trước, sau đó chạy lại file start.

5. Phần cho người hỗ trợ kỹ thuật

Phần này dành cho người cần kiểm tra sâu hơn bằng Terminal.

Kiểm tra nhanh phần mềm hệ thống
git --version
node --version
npm --version
python --version
uv --version
ffmpeg -version
espeak-ng --version
Setup thủ công backend
uv sync --frozen
uv pip install --force-reinstall vieneu==2.7.0 sea-g2p==0.7.5
uv run --frozen python -m backend.main
Setup thủ công frontend
cd frontend
npm ci --no-audit --no-fund
npm run dev -- -p 3000 -H 127.0.0.1
Kiểm tra backend đang sống
curl http://localhost:8000/health
curl http://localhost:8000/v1/status
